# Story 1.7 — Contexto de sessão com escola ativa no painel

| Campo | Valor |
|--------|--------|
| **Epic** | 1 — Fundação multi-tenant e acesso |
| **Prioridade** | Alta |
| **Depende de** | 1.2 |
| **Desbloqueia** | Execução correta de RLS por tenant ativo |

---

## User story

**Como** usuário com múltiplas escolas,  
**quero** selecionar e alternar escola ativa na sessão,  
**para** operar dados do tenant correto em cada ação.

---

## Contexto

- PRD §3.4 descreve login/seleção de escola.
- Ações server-side dependem do contexto ativo para RLS.

---

## Comportamento esperado

1. Persistir escola ativa na sessão/cookie segura.
2. Fornecer helper único para recuperar contexto no servidor.
3. Bloquear execução sem contexto válido.

---

## Critérios de aceite (testáveis)

1. Troca de escola atualiza dados exibidos no painel.
2. Server actions usam sempre a escola ativa.
3. Sem escola ativa, usuário é redirecionado para seleção.

---

## Fora do escopo

- SSO entre múltiplas organizações externas.
- Preferências avançadas por dispositivo.
