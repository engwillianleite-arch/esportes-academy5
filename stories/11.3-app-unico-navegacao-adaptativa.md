# Story 11.3 — Navegacao adaptativa por tipo de usuario no app unico

| Campo | Valor |
|--------|--------|
| **Epic** | 11 — App Esportes Academy: jornadas por tipo de usuario |
| **Prioridade** | Media/Alta |
| **Depende de** | 10.2, 10.3, 11.1, 11.2 |
| **Desbloqueia** | Escalabilidade de novas jornadas no app unico |

---

## User story

**Como** produto,  
**quero** que menu, home e atalhos do app se adaptem ao tipo de usuario ativo,  
**para** manter a experiencia simples mesmo com multiplas jornadas no mesmo aplicativo.

---

## Contexto

- O app unico so escala bem se a navegacao for contextual.
- A mesma base de autenticacao e layout precisa servir para responsavel, professor e futuros tipos.
- A plataforma precisa evitar poluicao visual e vazamento de funcionalidades.

---

## Comportamento esperado

1. Shell unico com configuracao de navegacao por tipo de usuario.
2. Itens de menu, dashboard e atalhos variam por contexto ativo.
3. Componentes compartilhados reaproveitam identidade visual e infraestrutura comum.
4. O app pode crescer para novos tipos sem criar outro produto.

---

## Critérios de aceite (testaveis)

1. Menu do responsavel nao mostra acoes de professor e vice-versa.
2. Mudanca de tipo de usuario recompõe menu e home corretamente.
3. Links profundos validam o contexto antes de abrir a tela.
4. Estrutura preparada para adicionar novos tipos sem reescrever a navegacao inteira.

---

## Fora do escopo

- Personalizacao por usuario final.
- Theme builder por escola dentro do app.
- White-label mobile completo.

---

## Definition of Done

- [ ] Estrategia de navegacao por contexto documentada.
- [ ] Limites entre shell compartilhado e areas especificas definidos.
- [ ] Stories dependentes referenciadas no backlog principal.
